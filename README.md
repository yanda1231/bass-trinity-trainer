# Bass Trinity Trainer 取扱説明書

ベースの練習アプリです。**メトロノーム＋指板の運指ガイド＋タイミング判定**が1画面に入っています。ブラウザで開くだけで動き、インストールは不要です。

- **パソコンのChromeまたはEdgeで開いてください**（スマホ・タブレットは今のところ未対応です。キー操作と音声入力がPC前提のため）
- アプリ本体：https://yanda1231.github.io/bass-trinity-trainer/
- 合言葉（パスワード）画面が出たら、配布者から聞いた合言葉を入れてください（一度通れば次からは聞かれません）

## クイックスタート

1. Chrome または Edge で上のURLを開く
2. キーとパターンを選び、**Space** でスタート／ストップ
3. メトロノームに合わせて練習
4. **正解の音を聴きながら練習したい人**は、Metronomeブロックの **Guide Volume** を上げる（お手本の音がガイドどおりに流れます。慣れたら音量を絞って卒業）
5. 楽器の音でタイミング判定したい人は下の「AUDIOモードと較正」へ

## シンプルモード ♪

右上の **♪ボタン** を押すと、計測系の画面が全部消えて**メトロノームと指板ガイドだけ**のすっきり表示になります。タイミング計測に興味がない人はこれだけでOK。もう一度押せば元に戻ります（設定は覚えています）。

## 判定の読み方

- **＋の数字＝LATE（拍より遅れて弾いた）／−の数字＝EARLY（先走った）**
- **Just**：ズレが判定幅（Judge Window。初期値±20ms）に収まった割合
- **Average**：全ヒットの平均ズレ。**Median**：ズレを並べたときの真ん中の値（1回の大外しに引きずられない、実力の中心値）
- **SD**：ばらつき。小さいほど毎回同じタイミングで弾けている。目安：10ms未満はかなり安定（プロの音楽家で4〜20ms程度と言われます）
- **Trend**：平均ズレの向き。判定幅の半分（最大±15ms）以内ならCENTER
- **判定幅（Judge Window）** は結果画面と履歴（HISTORY）の両方に表示されます。どの厳しさで測った記録なのかが後から分かります
- 練習画面の数字や履歴の見出しは、**マウスを重ねると意味の説明が出ます**。用語を覚える必要はありません

### 結果の記録を見る（HISTORY）

セッションが終わると結果画面が出ます。過去のセッションと見比べたいときは **HISTORY** ボタンを押してください。直近30件の履歴が、それぞれの判定幅・Medianつきで一覧できます（セッション中でなくてもいつでも開けます）。Hits欄は「検出した音の数／予定の音の数」です。

## AUDIOモードの入口（AUDIOチップ）

楽器の音で判定するには、まず音声入力をONにします。Audio Inputブロックの **AUDIOチップ**（NEXTなどと同じ、押すと点灯するスイッチ）を押すと、マイク／オーディオインターフェースの入力が始まります。もう一度押すとOFF。ONの間だけ、入力メーターや検出系の操作が生きてきます。

## AUDIOモードの音量合わせ（Boost / Auto Setup / Test）

- **Auto Setup**（Onset Detectionブロック）：静かに2秒＋普段の強さで6回弾くだけで、BoostとSense（感度）のおすすめ値を自動計算して提案します。**迷ったらまずこれ**。機材や部屋が変わったら1回やり直すだけ
- **Boost**（Audio Inputブロック）：アプリの中だけで入力を増幅します。機材やスピーカーの音量には影響しません。検出が鈍いときは、まずこれを上げて入力メーターを7〜8割へ
- **Test**（Onset Detectionブロック）：練習を始めずに検出を試せます。弾くとLEDが光り、隣の数字（検出スコア）で「あとどれくらいで検出されるか」が見えます

## AUDIOモードと較正（キャリブレーション）

パソコンの中を音が通るとき、目に見えない「配達の時間」がかかります。メトロノームの音が鳴るまで、あなたの音がアプリに届くまで、それぞれ数十ミリ秒（1000分の数十秒）の遅れがあります。この遅れを測って差し引かないと、ぴったりに弾いたのに「遅い」と判定されてしまいます。

**Loopback Calibration（較正）** は、オーディオインターフェースの出力と入力をケーブル1本で直結し、アプリがテスト音を16回鳴らして往復時間を正確に測る機能です。Auto Setupで音量を合わせたら、次はこれ。画面のチェックリストに従えば安全に進められます。

操作行は2行に分かれています。

1. **1行目：Start Loopback Cal** — これを押すと較正が始まります。テスト音が鳴り、往復時間を測ります
2. **2行目：APPLY ＋ Offset ms** — 測った値（Offset ms に出ます）を確定して使うときに **APPLY** を押します。手で微調整したい人は Offset ms の数値を直接入れてから APPLY でもOK

- 測った値はブラウザに保存されますが、**ブラウザを開き直すと数ミリ秒ずれることがあります。真剣に計測する日は、開いたら較正1回、が合言葉です**
- 較正しなくてもメトロノームとガイドはそのまま使えます（シンプルモード♪ならそもそも較正不要）

## Onset Detection（音の立ち上がり検出）

AUDIOモードのとき、アプリはあなたが弾いた「音の立ち上がり（オンセット）」を聴き取って、メトロノームとのズレを測ります。3つのスライダーは、ぜんぶ「どの音を本物の1音と認めるか」を決める門番の設定です。

| スライダー | 役割 | こんなときに動かす |
|---|---|---|
| **Sense**（感度） | 門番の厳しさ | 弾いたのにカウントされない→下げる ／ 弾いてないのに反応（Ghost）→上げる |
| **Refract**（休止時間） | 1音認めたあと次を受け付けない時間 | 1音が2回数えられる→上げる ／ 速いフレーズを取りこぼす→下げる |
| **Gate**（無音ゲート） | これより小さい音は門前払い | ノイズや弦のこすれで誤反応→上げる（-60くらい） ／ 小さい音も拾いたい→下げる |

### 検出の状態バッジ（CAL / UNCAL / RECAL / CHECK）

Onset Detectionの見出しには、較正の状態を示す小さなバッジが出ます。**マウスを重ねると、その状態の意味が表示されます**。

- **CAL**＝較正済み（遅れの補正が効いています）
- **UNCAL**＝未較正（較正すると判定が正確になります）
- **CHECK**＝測った値の確認待ち（APPLYを押すと適用されます）
- **RECAL**＝機材の構成が変わったので、較正のやり直しをおすすめします

### Advanced（普段は触らなくていい）

音の分析エンジンの内部設定です。困っていないなら初期値のままが正解。

- **Frame**：音を分析する「窓」の大きさ。大きいほど音色の変化に気づきやすいが、反応がのんびりになる
- **Hop**：分析の頻度。小さいほど時間の精度が上がるが、パソコンの負担が増える
- **Median pre / post**：「直前の音量の流れ」と比べるための参照範囲。ノイズの多い環境で判定がふらつくときの調整用
- いじって分からなくなったら、「設定を初期値に戻す」で全部初期値に戻せます（較正値は消えません）

## こまかい操作

- 使い方ガイドなどの画面は **ESCキー** か **画面の外側クリック** で閉じられます
- Metronomeブロックの **スピーカーアイコン** を押すと、クリックとガイド音を1小節だけ試し鳴らしして音量バランスを確認できます。ただし **Guide Volume やメトロノーム音をOFFにしているものは、試聴でも鳴りません**
- 判定の数字は **＋がLATE（遅れ）／−がEARLY（先走り）**。Averageには向きも表示されます

## 困ったときは

| 症状 | 対処 |
|---|---|
| マイク許可のダイアログが出ない | アドレスバーの鍵アイコン→マイクを「許可」に |
| 入力メーターが動かない | AUDIOチップがONか、入力デバイスとChannel（通常1ch）の選択を確認 |
| メーターは動くのに検出されない | **まずAuto Setupを実行**。手動なら Boost を上げてメーター7〜8割→足りなければSenseを下げる |
| 「入力が過大です」の赤警告 | 入力ゲインを下げてから「復帰」ボタン |
| 判定がいつも同じ方向にズレる | ループバック較正をやり直す（開いたら較正1回） |
| 表示がおかしい・設定が迷子 | 「設定を初期値に戻す」（較正・履歴は残ります） |
| 全部まっさらにやり直したい | 「全データを消去」（確認あり。較正・履歴・設定がすべて消えます） |

## ブラウザの音声補正について

パソコンのブラウザは、もともとビデオ通話向けに音を「聞きやすく」加工する機能を持っています。代表的なものが3つあります。

- **エコー除去（EC）**：スピーカーから回り込む音を消す
- **ノイズ抑制（NS）**：環境ノイズを削る
- **自動音量（AGC）**：入力の大きさを勝手に上げ下げする

これらは通話には便利ですが、**演奏タイミングの計測には邪魔になります**。音の立ち上がりを削ったり、音量を勝手に動かしたりして、ズレの測定を狂わせてしまうからです。そのため本アプリは、音声入力を取得するときに **この3つをすべてOFFにするよう要求します**。

ほとんどの環境ではOFFにできますが、まれにブラウザや機材の都合でOFFにできないことがあります。その場合だけ画面に警告が出ます。**警告が出たまま計測すると精度が落ちる**ので、可能なら別のブラウザや入力機器を試してください（警告が出ていなければ気にしなくて大丈夫です）。
