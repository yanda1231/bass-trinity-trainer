# 友人宅テスト手順

目的は「演奏の上手さ」を採点することではなく、ベース1音をアプリが1回として安定して拾えるかを確認することです。最初から設定を追い込まず、まず初期値の結果を残してください。

## 持ち物

- ベース、シールド、オーディオインターフェース、パソコン
- Chrome または Edge
- 可能ならループバック較正用のケーブル
- 公開URL: <https://yanda1231.github.io/bass-trinity-trainer/>

## 0. 最初の状態

1. 公開URLを開く
2. 「設定を初期値に戻す」を実行する、または Refract が **80 ms** になっていることを確認する
3. AUDIOをONにし、使う入力機器とChannelを選ぶ
4. 入力メーターが動き、赤い過大入力警告が出ないことを確認する

記録:

- パソコン:
- ブラウザ:
- オーディオインターフェース:
- 入力Channel:
- ベース／奏法:

## 1. 初期値の検出確認

Onset Detectionの **Test** を使います。練習はまだ開始しません。

1. 普通の強さで、十分な間隔を空けて10回弾く
2. 弱く10回、強く10回も弾く
3. 弾いていない状態で10秒待つ

奏法ごとに記録します。

| 奏法・強さ | 弾いた回数 | 検出回数 | 二重検出 | 備考 |
|---|---:|---:|---:|---|
| 指・普通 | 10 |  |  |  |
| 指・弱い | 10 |  |  |  |
| 指・強い | 10 |  |  |  |
| ピック・普通 | 10 |  |  |  |
| 無演奏10秒 | 0 |  |  |  |

ここでは、Sense・Refract・Gateをまだ動かしません。

## 2. Auto Setup

1. **Auto Setup** を開始する
2. 指示どおり静かに2秒待つ
3. 「普段の強さで、いつも通り4弦E（開放弦）を弾いてください」と表示されたら、案内が「音を止めてください」に変わるまで普段どおり弾く
4. 音を止め、表示が変わったら同じようにA・D・G弦を順番に測る
5. 提案された Boost と Sense を記録してから適用する
6. 「1. 初期値の検出確認」を普通の強さで再度行う

- 適用後の検出回数:

## 3. 実際のテンポで確認

初期値のRefract 80 msはそのままにして、4分音符で確認します。

1. BPM 60で1回練習する
2. BPM 120で1回練習する
3. BPM 200で1回練習する
4. 各結果画面の Hits（検出数／予定数）を記録する

## 4. ループバック較正

必要なケーブルがある場合だけ行います。

1. 画面の注意事項どおりに出力と入力を接続する
2. **Start Loopback Cal** を実行する
3. 表示された Offset ms と SD を記録する
4. 同じ条件でもう一度測り、大きく変わらないか確認する
5. 問題がなければAPPLYする

## 設定を動かす順番

1. まずAuto Setupで Boost と Senseを合わせる
2. 弾いていないのに反応する場合はGateを見直す
3. 1音が2回になる場合にだけRefractを上げる
4. BPM 200で取りこぼす場合にだけRefractを下げる

設定を変えた場合は、変更前後の値と結果を必ず一緒に記録してください。
